# v2 architecture

## session identity

`Session.id` is generated by the iOS app before the first message and is also passed through to the daemon and Claude as the `--session-id` value. The same identifier is therefore used locally and remotely from session creation onward.

This means `Session` does not need a separate `claudeSessionId` field.

- `Session.swift` — `id`, `envId?`, `path?`, `lastOpenedAt`, `title?`, `symbol?`, `skills?: [Skill]`, `agents?: [Agent]`
